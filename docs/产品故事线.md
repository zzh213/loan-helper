## 0. 一句话定位(电梯陈述)

**"小微贷管家是给小微企业主和普通人的'贷款翻译官 + 私人军师'——它把看不懂的银行黑话翻译成大白话,把'我到底能不能贷、能贷多少、找谁贷'的焦虑,变成一份看得懂、能照做的融资方案。"**

一句更短的版本:**"我们不放贷、不碰钱,只做小微和金融机构之间最难的那一环——让信息不对称的双方精准匹配。"**

---

## 1. 背景与问题:一场普通人打不赢的信息战

小微企业贡献了中国 60% 的 GDP、80% 的城镇就业,但"融资难、融资贵"是老问题。我在调研中发现,真正卡住小微企业主的,往往**不是没有贷款产品,而是三个"不知道":**

- **不知道自己能不能贷、能贷多少** —— 面对上百个产品无从下手,又怕申请被拒影响征信;
- **不知道该找谁贷、会不会被拒** —— 银行、担保、助贷信息不透明,容易被坑;
- **看不懂银行的专业话术** —— "年化""抵押率""征信""LPR",每个字都认识,连起来不懂。

**本质:这是一场普通经营者与金融机构之间的"信息不对称"。** 强监管、高专业门槛的金融领域,把最需要钱的人挡在了门外。

## 2. 目标用户与场景(Persona & JTBD)

| 用户 | 典型画像 | 核心任务(JTBD) |
|------|---------|----------------|
| **小微企业主 / 个体户**(主) | 开店、做电商、小工厂,年营收几十到几千万,财务不专业 | "帮我快速搞清楚我能贷多少、怎么贷最划算,并且我要看得懂" |
| **有融资需求的个人**(次) | 上班族、自由职业者,消费/经营/周转 | "我这条件能借到钱吗?哪个产品最适合我?" |
| **信贷经理 / 助贷**(潜在 B 端) | 每天要给客户出方案、备材料 | "帮我一天多签两单、少做重复劳动" |

**核心用户任务(JTBD)一句话:** "别让我做金融专家,直接告诉我答案,并且我要看得懂、能照做。"

---

## 3. 关键洞察

> **同类产品都在比"谁的产品库大、算法准",但用户真正的痛点是'看不懂'和'不敢试'。**
> 谁能把"专业"翻译成"人话",谁就赢得信任——而信任,才是金融产品的第一入口。

这个洞察决定了产品的**灵魂功能是「说人话版解读」**,而不是又一个冷冰冰的利率计算器。

---

## 4. 解决方案:一条主线串起所有功能

**主线:破除信息不对称 → 用大白话建立信任 → 手把手帮用户走完融资。**

| 层次 | 功能 | 解决哪个"不知道" |
|------|------|----------------|
| **看得懂** | 💬 说人话版解读 + 名词小课堂 | 看不懂话术 |
| **算得清** | 双引擎智能推荐(企业贷 / 个人贷)、8 维风控评分、方案分层对比 | 能不能贷、贷多少 |
| **办得成** | 材料清单、银行成品材料(PDF/Word)、预约信贷经理、申请进度追踪 | 找谁贷、怎么办 |
| **拿实惠** | 补贴政策匹配 + 隐藏贴息(区县/园区级) | 别漏掉政府扶持 |
| **敢信任** | 大白话 + 诚实的数据来源标注 + 政府担保增信方案 | 会不会被坑 |

**设计取舍案例(体现产品 sense):**
- **"精简版 + 查看更多"**:调研发现一次性铺开全部结果会造成信息过载,于是首屏只给"一句话结论 + 最优方案",其余收进折叠区——**降低认知负荷 > 展示功能全面**。
- **"常用按钮 + 更多功能"**:9 个操作按钮精简为 3 个常用 + 折叠——**减法就是产品力**。
- **大模型的克制使用**:核心推荐用确定性规则(稳定、零成本、可解释),只在开放问答场景接入大模型(GLM-4-Flash)——**不为了 AI 而 AI**。

## 5. 指标体系与数据看板(产品思维的硬证据)

> "一个产品经理的高光时刻不是'我做了功能',而是'我怎么知道它有用'。"

为此我搭建了**轻量埋点系统 + 管理后台数据看板**,把产品从"我觉得"升级为"数据说":

**北极星指标:** 完成融资匹配并留资的有效用户数。

**转化漏斗(6 步):**
```
访问 → 开始填表 → 提交测算 → 看到方案 → 展开完整详情 → 留资/预约
```
看板会**自动标出流失最严重的一环**,直接指向下一步该优化哪里。

**辅助指标:**
- 表单转化率、留资转化率、独立访客数;
- 功能使用排行(哪些功能真有人用 → 数据驱动的功能取舍);
- 每日访问/提交趋势。

**这套看板本身就是我"会定义指标、用数据驱动决策"的作品。**

## 6. 数据驱动的迭代案例(可讲成一个完整故事)

> 举例叙事(可结合真实数据替换):
> "上线后看漏斗,发现『提交测算 → 看到方案』流失最大,进一步分析是**结果页信息过载**,用户被吓退。
> 于是我做了『精简版 + 查看更多』改版,把首屏聚焦到一句话结论。
> 同时用埋点验证:改版后『展开完整详情』的点击率说明有多少用户真的想看深度内容,
> 而『说人话版解读』的名词小课堂点击量,验证了'看不懂'这个痛点是否真实存在。"

这个闭环展示了 **提出假设 → 埋点验证 → 迭代 → 再验证** 的完整产品方法论。

## 7. 商业模式(能讲清楚怎么转)

**主模式:助贷获客(CPA/CPS 分成)**
- 用户免费使用 → 产生"有真实意向 + 已完成画像"的高质量线索 → 分发给银行/持牌助贷机构 → 按有效对接(CPA)或成功放款(CPS,放款额 1–3%)收费。
- **合规护城河:不放贷、不碰资金、不做资金池**,只做撮合匹配,规避金融牌照风险。

**第二曲线:SaaS 工具版(To B)**
- 同一套引擎面向信贷经理/小助贷公司,做展业提效工具,按账号订阅收费。

**为什么组合成立:** 先用 To C 免费产品跑流量与数据,再用真实申请数据反哺风控与 To B 工具,形成**数据飞轮**。

---

## 8. 竞争壁垒:数据飞轮

```
更多用户 → 更多真实申请数据 → 风控模型越校准越准(已内置 calibrate 机制)
        → 匹配更精准 → 转化更高 → 吸引更多机构与用户 → 循环
```
产品里已经埋下技术支点:**用历史真实通过率反向校准各行业方案的本地通过率**——这就是"越用越准"的护城河雏形。
