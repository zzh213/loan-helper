## 一、项目概述(30 秒讲清楚)

**小微贷管家**是我在实习期间独立完成的一款智能贷款服务产品。用户填写自身情况(行业、营收、征信、抵押等),系统在几秒内智能匹配贷款产品、评估风控、给出最优方案,并**用大白话把专业内容讲清楚**,同时匹配可申报的政府补贴。

- **一句话定位:** 给小微企业主和普通人的"贷款翻译官 + 私人军师"。
- **核心价值:** 破除普通经营者与金融机构之间的信息不对称。
- **交付成果:** 一个真实上线、可安装(PWA)、含 AI 助手与数据看板的完整 Web 产品。

**关键数据(截至结项):**

## 二、我发现的问题(为什么做这个)

小微企业贡献了全国 60% 的 GDP、80% 的城镇就业,但"融资难"是老问题。我在调研中发现,真正卡住用户的**不是没有产品,而是三个"不知道":**

1. **不知道自己能不能贷、能贷多少** —— 面对上百个产品无从下手,又怕被拒影响征信;
2. **不知道该找谁贷、会不会被拒** —— 银行/担保/助贷信息不透明;
3. **看不懂银行的专业话术** —— "年化""抵押率""征信",每个字都认识,连起来不懂。

**本质是一场普通人打不赢的"信息不对称"。** 这就是产品要解决的核心痛点。

---

## 三、我做了什么(产品方案)

我用**一条主线**串起所有功能:**破除信息不对称 → 用大白话建立信任 → 手把手帮用户走完融资。**

| 层次 | 关键功能 | 解决哪个"不知道" |
|------|---------|----------------|
| 看得懂 | 💬 说人话版解读 + 名词小课堂 | 看不懂话术 |
| 算得清 | 双引擎推荐(企业/个人)、8 维风控评分、方案分层对比 | 能不能贷、贷多少 |
| 办得成 | 材料清单、银行成品材料(PDF/Word)、预约信贷经理、进度追踪 | 找谁贷、怎么办 |
| 拿实惠 | 补贴政策匹配 + 区县/园区级隐藏贴息 | 别漏掉政府扶持 |
| 敢信任 | 大白话 + 诚实的数据来源标注 + 政府担保增信 | 会不会被坑 |

## 四、我是怎么思考的(产品取舍,答辩重点)

功能多不难,**难的是取舍**。以下是最能体现产品思维的三个决策:

**1. 结果页"精简版 + 查看更多"——降低认知负荷 > 展示功能全面**
最初把全部结果一次性铺开,信息过载会吓退用户。我改为首屏只给"一句话结论 + 最优方案",其余(风控、对比、补贴等)收进折叠区。**克制地展示,才是对用户负责。**

**2. 「说人话版解读」——把"专业"翻译成"信任"**
我意识到同类产品都在比"谁算得准",但用户真正的痛点是"看不懂、不敢试"。于是我做了大白话解读——它不是一个普通功能,而是整个产品的**情感内核和差异点**。

**3. 大模型的克制使用——不为了 AI 而 AI**
核心推荐用确定性规则(稳定、零成本、可解释、结果可复现),只在开放问答场景接入大模型(GLM-4-Flash)。**在合适的地方用合适的技术,而不是堆砌热点。**

## 五、带来了什么价值(用数据说话)

我没有停留在"我觉得有用",而是**亲手搭建了一套埋点系统 + 数据看板**来验证价值。

**北极星指标:** 完成融资匹配并留资的有效用户数。

**转化漏斗(6 步):** 访问 → 开始填表 → 提交测算 → 看到方案 → 展开详情 → 留资/预约。
看板会**自动标出流失最严重的一环**,直接指向下一步该优化哪里。

**辅助指标:** 表单转化率、留资转化率、功能使用排行、每日趋势。

> 演示环境下,产品从访问到看到方案的转化约 40%,漏斗最大流失点出现在"提交测算→留资"环节——这正是下一步优化的靶点。
> (真实数据以答辩现场看板为准)

**这套看板本身,就是我"会定义指标、用数据驱动决策"的能力证明。**

## 六、一个完整的迭代案例(体现方法论)

以"结果页改版"为例,展示我的完整产品闭环:

1. **发现问题:** 观察到结果页信息量过大,担心用户流失;
2. **提出假设:** 首屏聚焦核心结论能降低认知负荷、提升深度浏览;
3. **落地方案:** 重构为"精简版 + 查看更多"折叠结构;
4. **埋点验证:** 上线埋点追踪"展开详情"点击率、名词小课堂点击量,验证用户是否真的想看深度内容、"看不懂"痛点是否真实;
5. **持续迭代:** 根据漏斗数据决定下一步优化靶点。

**这是一个"提出假设 → 埋点验证 → 迭代 → 再验证"的完整数据驱动闭环。**

## 八、商业模式(能讲清楚怎么转)

- **主模式——助贷获客(CPA/CPS 分成):** 免费产品产生高质量线索 → 分发给持牌机构 → 按有效对接或成功放款分成。**不放贷、不碰资金,只做撮合,规避牌照风险。**
- **第二曲线——SaaS 工具版:** 同一套引擎面向信贷经理做展业提效工具,按账号订阅。
- **护城河:** 先用 To C 产品跑流量与数据,再用真实数据反哺风控与 To B 工具,形成数据飞轮。

## 九、成长与反思(答辩加分项)

**我的收获:**
- 从"做功能"到"做产品"——学会用一条主线做取舍,而非堆砌功能;
- 建立了"数据驱动"的习惯——任何功能都要能回答"我怎么知道它有用";
- 打通了产品全流程——第一次独立把一个想法做到真实上线可用。

**我的不足与改进:**
- 数据真实性:当前产品/补贴数据多为示意性整理,商业化前需对接真实数据源并做合规审查;
- 缺少真实用户验证:下一步应做真实用户访谈,用定性反馈补充定量数据;
- 转化闭环未打通:留资后的机构对接与结算流程尚未落地。
